Lynx 3.8 已经正式发布！

延续 [2026 路线图更新](/blog/lynx-open-source-roadmap-2026)中提到的规划，Lynx 3.8 是我们更快月度发布节奏下的首个版本。本次版本在 Android 上带来了基础 **WebAssembly** 支持，并新增 `<frame>` 自适应尺寸、基于行数的自适应字号范围、HarmonyOS SessionStorage，以及原生 API 和 DevTool 更新。

## Android 上的 WebAssembly

本次发布通过 PrimJS 运行时集成，在 Android 上引入了基础 [WebAssembly](/guide/scripting-runtime/webassembly) 支持。Lynx 应用现在可以在后台线程加载和执行 WebAssembly 模块，让计算密集型逻辑在 JavaScript 之外获得新的实现选择。

当前实现主要覆盖常见的构造函数式使用方式：

```js
const module = new WebAssembly.Module(wasmBytes);
const instance = new WebAssembly.Instance(module, imports);

const result = instance.exports.add(1, 2);
```

当前支持覆盖常见场景下的 `WebAssembly.Module` 和 `WebAssembly.Instance`，以及 `WebAssembly.Memory`、`WebAssembly.Table` 和 `WebAssembly.Global`。

## UI 与平台更新

本次发布也带来了多项更新，以改进嵌入式 UI 组合、文本自适应和平台覆盖。

在组件级 UI 方面，可以参考我们最近发布的 [lynx-ui](/zh/blog/lynx-ui)，这是一套开箱即用的组件库，可用于构建跨平台 Lynx 界面。

### `<frame>` 自适应尺寸与加载指标

[`<frame>`](/api/elements/built-in/frame) 现在可以通过新的 `auto-width` 和 `auto-height` 属性，根据嵌入 Lynx 页面的内容确定自身尺寸，减少宿主侧手动计算尺寸的需要。

```tsx
<frame src={url} auto-width auto-height />
```

它也增强了启动和可观测性支持，包括预设布局、更好的 `data` 与 `global-props` 传递、通过 `layoutchange` 上报固有尺寸，以及 Android/iOS 帧加载指标。

### 基于行数的自适应字号范围

新的 [`-x-auto-font-size-line-ranges`](/api/css/properties/-x-auto-font-size-line-ranges) CSS 属性可以根据实际渲染行数定义不同的字号搜索范围。配合 `-x-auto-font-size: true` 使用：

```css
.title {
  -x-auto-font-size: true;
  -x-auto-font-size-line-ranges:
    line-range(1, 18px, 22px), line-range(2, 16px, 18px),
    line-range(3 to infinity, 14px);
}
```

这适用于标题、卡片和信息流等布局，让单行、双行和多行文本获得不同的适配效果。

### 平台覆盖更新

macOS 和 Windows 现在支持 [`enable-exposure-ui-clip`](/api/elements/built-in/view.html#enable-exposure-ui-clip)，让开发者可以在复杂布局中更精细地控制曝光计算。

平台覆盖也进一步改进，HarmonyOS `<view>` 新增[关键帧动画](/api/css/properties/animation)管线支持。

## 原生 API

### LynxViewGroup 中的引擎复用

在 Android 上，`LynxViewGroup` 现在可以为同一个组创建出的 `LynxView` 缓存并复用 `LynxEngine`。当应用反复基于同一个 template bundle 创建视图时，例如多页面或嵌入式视图场景，这一能力会很有用。

我们发现，在很多业务场景中，用户会反复离开并重新进入同一个 Lynx 页面。在这种情况下，可以缓存 `LynxEngine`，避免大内存对象反复析构和创建；同时也可以复用缓存引擎中已经完成的工作，包括已经构建完成的 UI 树，从而加速下一次页面重入时的渲染。

应用无需直接管理 `LynxEngine`。创建一个启用引擎缓存的组，将该组传给每个 `LynxViewBuilder`，并在不再需要时释放组：

```java
ILynxViewGroup group = new LynxViewGroupBuilder()
    .setContext(context)
    .setTemplateBundle(templateBundle)
    .setGlobalProps(globalProps)
    .setEnableCacheEngine(true) // 启用缓存。
    .build();

LynxView first = new LynxViewBuilder()
    .setLynxViewGroup(group)
    .build(context);

// 稍后，使用同一个 group 创建的另一个 LynxView 可以复用缓存的 engine。
LynxView second = new LynxViewBuilder()
    .setLynxViewGroup(group)
    .build(context);

group.release();
```

### 资源加载回调

`LynxViewClientV2` 新增 `onResourceLoaded` 回调，使客户端可以在原生侧观察资源加载过程。这为宿主应用收集资源耗时、成功与失败信息提供了更清晰的入口。

### HarmonyOS 上的 SessionStorage

HarmonyOS 现在支持 [SessionStorage API](/api/lynx-api/lynx/lynx-get-session-storage)，可用于跨页面数据共享。前端代码可以在 HarmonyOS 上使用已有的 `lynx.setSessionStorageItem`、`lynx.getSessionStorageItem`、`lynx.subscribeSessionStorage` 和 `lynx.unsubscribeSessionStorage` API，管理 session 级的跨页面状态。

## DevTool 更新

DevTool 协议和调试能力改进包括：

- MTS 调试支持 pre-execute 和 pre-decode `MTSRuntime`。
- CDP 支持新增 `DOM.describeNode`。
- `DOM.getDocument` 现在支持 `depth` 参数。
- DevTool 可以通过 CDP Performance domain 查询 `PerformanceEntry` 数据。
