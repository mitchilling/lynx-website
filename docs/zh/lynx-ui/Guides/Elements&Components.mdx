x-element 提供的是原生元件，它们由底层 native 实现，由客户端提供，无需其他引用步骤，前端可直接使用。 lynx-ui 组件库提供前端组件，它是基于元件之上，使用 ReactLynx 构建的复合视图，提供了更加丰富的渲染及交互能力。

## 为什么在元件之上需要组件？

我们想用前端组件补足 native 元件的适配能力，建设高性能、native-like、兼容性好的 Lynx 组件生态。

同平台的 UI 特性在行为、API
乃至设计理念上都有显著差异，尤其是高阶功能更是如此。跨平台框架需要尽可能地收敛这些差异，Lynx
也不例外。

前端组件将在这些底层原子化 API 庞大的数量基础上，梳理它们的表现和限制，在前端侧达到最终的一致。
