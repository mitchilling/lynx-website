# Lynx DevTool MCP

Lynx DevTool MCP 是一个 [Model Context Protocol (MCP) 服务器](https://modelcontextprotocol.io/docs/learn/server-concepts)，它让 coding agent 能够 _**控制、操作和预览**_ Lynx 页面。

## 开始使用

将以下配置添加到 MCP 客户端：

```json
{
  "mcpServers": {
    "lynx-devtool": {
      "command": "npx",
      "args": ["-y", "@lynx-js/devtool-mcp-server@latest"]
    }
  }
}
```

## 使用方法

使用 Lynx DevTool MCP 服务器就像使用 DevTool 一样，需要连接设备并打开 LynxExplorer 等应用程序。

### 元素

就像 DevTool 中的"元素"标签页一样，
Lynx DevTool MCP 服务器中有各种工具（前缀为 `CSS_*` 或 `DOM_*`）来检查元素树。

:::note
所有这些工具都是只读的。修改源代码以对元素树进行更改。
:::

### 控制台

就像 DevTool 中的"控制台"标签页一样， coding agent 可以使用 Lynx DevTool MCP 服务器读取控制台中的所有消息及其堆栈跟踪。

### 源代码

就像 DevTool 中的"源代码"标签页一样，Lynx DevTool MCP 服务器可以读取所有加载的 JavaScript 源代码。

这对于 coding agent 在控制台堆栈跟踪中找到相应的代码很有用。

### 交互

coding agent 还可以使用 Lynx DevTool MCP 服务器与 Lynx 页面进行交互，执行点击或拖动等操作。

### 截图

Lynx DevTool MCP 服务器还提供工具来截取当前 Lynx 页面的屏幕截图，使用多模态模型来理解屏幕截图。
