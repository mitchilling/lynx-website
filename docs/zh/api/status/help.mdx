# API 状态面板使用指南

:::tip
本使用指南由 AI 辅助生成。如果您发现任何不准确之处，欢迎提交 Pull Request 帮助我们改进。
:::

本指南介绍如何使用 Lynx API 状态面板来了解不同平台的 API 兼容性。

![API 状态面板概览](/api-status-help/api-status-main.png)

---

## 1. 选择平台

在面板顶部，您会看到**平台选择器**，可以在不同平台之间切换查看各自的 API 覆盖率。

### 原生平台

- **Android** - 移动端 Android 平台
- **iOS** - Apple iOS 平台
- **Harmony** - HarmonyOS 平台
- **Web** - Web Lynx 实现

每个平台按钮显示其名称、图标和当前 API 覆盖率百分比。

### Clay 平台

点击 **Clay** 切换按钮以显示更多 Clay 平台选项：

![Clay 平台展开](/api-status-help/api-status-clay-platforms.png)

Clay 平台包括：

- **Clay (Android)** - 运行在 Android 上的 Clay
- **Clay (iOS)** - 运行在 iOS 上的 Clay
- **Clay (macOS)** - 运行在 macOS 上的 Clay
- **Clay (Windows)** - 运行在 Windows 上的 Clay

---

## 2. 搜索、筛选和查看 API

### 搜索框

使用搜索框按名称或描述快速查找特定 API。搜索不区分大小写，支持部分文本匹配。

**示例：**

- 搜索 `image` 查找所有图片相关 API
- 搜索 `scroll` 查找滚动功能
- 搜索 `animation` 查找动画 API

### 分类筛选

第一个下拉菜单允许您按分类筛选 API：

- **全部** - 显示所有分类
- **ELEMENT** - 元素相关 API
- **CSS** - CSS 属性支持
- **API** - JavaScript API 方法

### 状态筛选

第二个下拉菜单按选定平台的支持状态筛选 API：

- **全部** - 显示所有 API
- **已支持** - 仅显示已支持的 API
- **未支持** - 仅显示未支持的 API

### API 列表

API 列表以两列网格布局显示所有匹配的 API。每个 API 项显示：

- **分类标签**（例如 `ELEMENT`、`CSS`、`API`）
- **API 名称**（例如 `commonality`、`image.loop-count`）
- **颜色编码**：
  - 🟢 **绿色背景** - API 在选定平台上已支持
  - 🔴 **红色背景** - API 在选定平台上未支持

默认情况下，为了性能只显示前 100 个结果。点击**「显示全部」**以显示所有匹配结果。

### API 详情抽屉

点击任意 API 项以打开详细兼容性抽屉：

![API 详情抽屉](/api-status-help/api-status-drawer.png)

抽屉显示兼容性表格，包含：

- **行** - 每个子 API 或属性
- **列** - 所有平台（Android、iOS、HarmonyOS、macOS、Windows、Web）

| 图标       | 含义             |
| ---------- | ---------------- |
| ✓ 带版本号 | 从该版本开始支持 |
| ✗ No       | 不支持           |
| ?          | 支持状态未知     |

点击**「查看源文件」**链接可在 GitHub 仓库中查看原始兼容性数据。

## 3. 查看平台覆盖率和趋势

底部区域显示平台统计信息和历史趋势：

![覆盖率卡片和趋势图表](/api-status-help/api-status-coverage-trend.png)

### 覆盖率卡片（左侧）

- **百分比** - 总体 API 覆盖率（例如 87%）
- **分数** - 精确计数（例如 945 / 1,092）
- **进度条** - 带有平台特定颜色的可视化表示

### 趋势图表（右侧）

- **X 轴** - Lynx 版本（例如 2.14 → 3.4）
- **Y 轴** - API 覆盖率百分比
- **折线** - 随时间推移的覆盖率进展

这有助于您了解 API 支持在各版本中的改进情况。

### 分类表格

**Coverage** 部分提供按 API 分类的细分：

![分类表格展开视图](/api-status-help/api-status-category-expanded.png)

| 列     | 描述                               |
| ------ | ---------------------------------- |
| 分类   | API 分类名称（例如 Elements、CSS） |
| 覆盖率 | 已支持 API 的百分比和计数          |
| 缺失   | 未支持 API 的数量（如有）          |

**高亮模式：**

- **Highlight Good**（绿色模式）- 强调高覆盖率分类
- **Highlight Gap**（红色模式）- 强调低覆盖率分类

点击任意分类行以展开并查看各个 API。未支持的 API 会以红色高亮显示，便于识别。

## 4. 最近添加的 API

**最近添加** 部分列出了最近添加到 Lynx 的 API，按版本分组。

![按版本分组的最近添加 API](/api-status-help/api-status-recently-added.png)

每个版本组显示：

- **版本号**（例如 v3.5、v3.4、v2.18）
- 选定平台的 **API 数量**
- 该版本中添加的 **API 列表**

这对于跟踪每个 Lynx 版本中的新功能非常有用。

---

## 提示

1. **先选择平台** - 首先选择您的目标平台以查看相关覆盖率数据。
2. **使用搜索查找特定 API** - 如果您知道 API 名称，搜索比滚动更快。
3. **查看抽屉获取详情** - 兼容性表格显示版本特定信息。
4. **关注趋势** - 使用趋势图表了解覆盖率随时间的改进。
5. **启用 Clay 用于桌面端** - 如果您正在构建跨平台桌面应用，请启用 Clay 平台。
