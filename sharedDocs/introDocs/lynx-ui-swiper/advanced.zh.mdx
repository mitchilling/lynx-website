## 进阶用法

当默认滑动布局不够用时，可以使用进阶 API。`customAnimation` 让每个
item 根据滑动进度更新样式，`mode` 决定 Swiper 是负责 item 的位置，还是
把位置完全交给动画函数。

### 理解 `customAnimation`

```
type customAnimation = (value: number, index: number) => CSSProperties
```

`customAnimation` 是一个主线程函数。Swiper 滚动时，每个 `SwiperItem`
都会收到自己相对当前选中项的距离，并根据函数返回值更新当前帧的样式。

其中，函数入参`value`表示这个`<SwiperItem>`相对于选中`<SwiperItem>`的距离。

在这个例子中，`<Swiper>`的进度为 2，于是：

- `<SwiperItem>-2`收到的`value`是 0
- `<SwiperItem>-1`收到的`value`是 -1
- `<SwiperItem>-3`收到的`value`是 1

更进一步，`<Swiper>`的进度被更新到 3 时，会有：

- `<SwiperItem>-2`收到的`value`是 -1
- `<SwiperItem>-1`收到的`value`是 -2
- `<SwiperItem>-3`收到的`value`是 0

可以把这个值作为插值输入来理解：`0` 表示当前选中项，`-1` 表示前一项，
`1` 表示后一项。

### `customAnimation`的实践

当 Swiper 仍然使用默认布局，而 item 只需要缩放、透明度或旋转等视觉效果
时，使用这种方式即可。

## 理解 `mode`

`<Swiper>`支持`normal`和`custom`两种`mode`。他们的区别如下：

`mode='normal'`

- `<SwiperItem>`按照方向依次排列
- 根据 `modeConfig` 提供了居中，向右对齐等能力
  `mode='custom'`

- `<SwiperItem>` 的位置需要通过 [`customAnimation`](#理解-customanimation)指定
- 自定义的能力更高

总结起来

- `normal` mode 处理了`<SwiperItem>`的位置，比较适合横滑的场景，使用起来更方便
- `custom` mode 需要对位置进行手动计算，但是扩展性更高

### `mode='custom'` 时 `customAnimation` 的实践

当 item 的位置本身就是设计的一部分时，使用 `custom` mode。在这种模式
下，动画函数需要同时返回位置和视觉样式。
